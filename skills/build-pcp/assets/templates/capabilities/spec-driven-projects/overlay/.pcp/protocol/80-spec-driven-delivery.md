# Spec-driven delivery

- Use one bounded specification for a project, feature, migration, or other defined outcome.
- Separate outcome and acceptance criteria from implementation sequencing.
- Derive the plan and structured workstreams from the accepted specification.
- Keep assumptions, exclusions, risks, and human decision gates explicit.
- Implement in coherent verified units and update the living plan as evidence changes.
- Completion requires every acceptance criterion to have current evidence; document partial or external completion honestly.
- A project specification is one project record, not the universal definition of the enclosing portfolio or organization.
