# Domain and invariants

## Vocabulary

- Define project-specific terms that an agent must interpret consistently.
- Prefer the project's established language over invented abstractions.

## Invariants

- Record rules that must remain true across code, data, documents, workflows, or releases.
- Tie each invariant to observable evidence, enforcement, or a test where possible.

## Failure and risk

- Identify irreversible operations, security boundaries, and high-cost failure modes.
- Keep temporary incidents in work records; retain only enduring lessons here.
